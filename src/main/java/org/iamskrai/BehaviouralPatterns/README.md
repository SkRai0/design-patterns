## 🚀 Behavioural Design Patterns

Behavioural patterns focus on **how objects communicate and collaborate** to complete tasks.

These patterns help you:
- 🔄 organize interactions between objects,
- 🧼 avoid condition-heavy and tightly coupled logic,
- 🧩 design systems that are easier to extend and maintain.

## 🧩 Folder Structure

Each pattern typically contains:
- `Problem/` -> direct or tightly coupled implementation
- `Solution/` -> pattern-based implementation
- `README.md` -> concise explanation of problem and solution approach

## 🎯 Learning Outcome

By practicing behavioural patterns, you get better at:
- encapsulating varying behavior,
- reducing coupling between components,
- making runtime behavior flexible,
- designing cleaner communication flow between objects.
