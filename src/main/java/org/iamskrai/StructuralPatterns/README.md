## 🚀 Structural Design Patterns

Structural patterns focus on **how classes and objects are composed** to build flexible and maintainable systems.

These patterns help you:
- 🧩 connect incompatible interfaces cleanly,
- 🔧 reduce tight coupling between collaborating components,
- 📐 keep object composition simple, reusable, and extensible.

## 📚 Patterns Covered

- [Adapter Pattern](AdapterPattern/README.md)

## 🧩 Folder Structure

Each pattern typically contains:
- `Problem/` -> incompatible or tightly coupled structure
- `Solution/` -> pattern-based structural composition
- `README.md` -> quick explanation of problem and why pattern helps

## 🎯 Learning Outcome

By practicing structural patterns, you get better at:
- integrating legacy/incompatible APIs,
- composing objects with cleaner boundaries,
- minimizing ripple effects from implementation changes,
- designing adaptable system structures.
