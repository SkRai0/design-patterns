## 🚀 Creational Design Patterns

Creational patterns focus on **how objects are created** in a flexible, reusable, and maintainable way.

Instead of creating objects directly everywhere with `new`, these patterns help you:
- 🧠 decouple object creation from business logic,
- 🔁 reuse creation logic,
- 📈 scale easily when new object types are added.

## 🧩 Folder Structure

Each pattern typically contains:
- `Problem/` -> naive or tightly coupled implementation
- `Solution/` -> pattern-based clean implementation
- `README.md` -> quick explanation of problem and why pattern helps

## 🎯 Learning Outcome

By practicing creational patterns, you get better at:
- managing object lifecycle,
- reducing construction complexity,
- following Open/Closed Principle,
- designing extensible object creation flows.
